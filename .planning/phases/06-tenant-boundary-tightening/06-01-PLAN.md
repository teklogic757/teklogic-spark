<objective>
Establish an explicit dashboard access layer that narrows service-role usage and removes the broad inline trust boundary.

Purpose: The current dashboard page creates both user-scoped and admin clients, then mixes them across multiple reads with inline casts. This plan moves that logic into one helper layer, limits service-role fallback to the org-resolution case, and replaces the most obvious `as any` escapes with typed contracts.
Output: A shared dashboard access helper and a cleaner dashboard page that uses least-privilege queries wherever possible.
</objective>
