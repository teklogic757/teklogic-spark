<objective>
Extract the first shared submission-domain layer so the desktop and mobile actions stop duplicating the same setup pipeline.

Purpose: The current desktop and mobile submit actions each perform rate limiting, user/org resolution, AI payload construction, and evaluation setup inline. This plan creates a shared server-side helper module with typed contracts for that preparation work and updates both actions to delegate to it without changing their user-facing behavior.
Output: A new `submit-flow` helper module plus leaner desktop and mobile actions that no longer duplicate the early submission orchestration.
</objective>
